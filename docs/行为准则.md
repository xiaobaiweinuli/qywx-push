# 企业微信通知服务（QYWX Push）开源项目行为准则
## 准则目的与核心价值观
### 目的
本行为准则旨在为企业微信通知服务（QYWX Push）开源项目的所有参与者，构建一个尊重、包容、安全且高效的协作环境。本项目为个人独立开发项目，准则明确了协作中的行为规范、权利与责任，确保所有参与者都能在公平、友善的氛围中贡献力量。

### 核心价值观
- **以人为本**：每一位参与者都是项目的宝贵财富，应得到人道、同理心的对待与尊重。
- **诚信合规**：坚守法律底线与职业操守，所有协作行为均需符合适用的法律法规与公序良俗。
- **开放协作**：欢迎所有认同项目使命的参与者，鼓励跨背景、跨经验的交流与合作。
- **多元包容**：尊重不同的主张、观点与经历，主动消除排斥与偏见，打造人人乐于加入的协作空间。

## 适用范围
本行为准则适用于所有参与 QYWX Push 项目的个人与群体，包括但不限于项目维护者（个人开发者）、代码贡献者、文档编写者、issues 讨论参与者、使用者等。  
适用场景仅涵盖项目线上协作活动，包括：
1. GitHub 仓库（代码提交、Pull Request、issues、讨论区、Wiki）；
2. 项目官方沟通渠道（仅通过 GitHub 仓库相关功能进行）。  
*注：本项目为个人独立开发，无其他沟通渠道，不组织任何线下活动。*

## 核心行为规范
### 1. 尊重包容
- 尊重每一位参与者的人格、观点与贡献，即使存在分歧，也应针对论题本身进行理性探讨，而非攻击个人。
- 主动接纳不同背景、经验、技术水平的参与者，避免因种族、民族、种姓、肤色、年龄、外貌、残障、性别、性别认同、性取向、语言、宗教、国籍、社会经济地位等身份特征产生歧视或偏见。

### 2. 合法合规
- 所有参与项目相关活动的行为，均需遵守中华人民共和国及参与地的法律法规、行业规章，严禁任何违法违规操作（如提交恶意代码、泄露他人隐私、侵犯知识产权等）。
- 遵守开源社区通用规范，尊重项目的开源协议及相关政策。

### 3. 开放友善
- 以开放的心态接纳新参与者，耐心解答疑问，主动分享技术经验与项目知识，助力他人融入协作。
- 沟通中保持礼貌与友善，避免使用侮辱性、攻击性、嘲讽性语言，即使发生矛盾，也应通过理性方式解决，杜绝人身攻击。

### 4. 协作共赢
- 明确自身对项目的责任，对提交的代码、文档等贡献内容负责，确保质量与安全性。
- 尊重他人的劳动成果，引用或借鉴第三方内容时必须注明来源；乐于配合代码评审，积极接受建设性反馈，共同推动项目优化。

### 5. 谨言慎行
- 维护专业的协作氛围，禁止任何形式的骚扰、歧视、欺凌或排他性行为，包括但不限于：
  - 发表歧视性笑话、种族主义或性别歧视言论；
  - 进行不受欢迎的性关注或相关暗示；
  - 反复骚扰、恐吓他人或恶意挑衅；
  - 发布侮辱性、贬低性的个人评价。
- 谨慎使用社交媒体及公共平台，避免发表损害项目声誉、攻击其他参与者的言论。

## 禁止行为
除上述规范外，以下行为明确禁止，任何违反均将受到相应处理：
1. **身份欺诈**：冒充项目维护者（个人开发者）、贡献者或其他相关人员，或伪造身份逃避违规责任。
2. **侵犯权益**：未经许可分享他人隐私信息（如联系方式、身份信息），或侵犯第三方知识产权、商业机密。
3. **危害项目安全**：提交恶意代码、散播项目未公开漏洞、泄露核心技术机密等损害项目及使用者权益的行为。
4. **违规推广**：在项目仓库、issues 等协作场景发布无关的商业广告、营销内容。
5. **利益冲突隐瞒**：参与项目贡献或决策时，若存在可能影响公正的利益关系（如商业合作关联），未主动披露。
6. **报复行为**：对举报违规行为的参与者进行打击报复，或阻碍违规调查的正常进行。

## 投诉与报告流程
### 报告渠道
参与者若遭遇或发现违反本准则的行为，需通过以下唯一官方渠道举报：
- **GitHub issues 举报**：在项目 GitHub 仓库（https://github.com/xiaobaiweinuli/qywx-push/issues/new） 提交 issues，标题需标注【违规举报】，正文详细说明事件时间、场景、相关链接、证据及涉及人员。  
*注：本项目仅接受 GitHub issues 这一种举报渠道，不接受邮件或其他非官方方式的举报。*

### 报告处理流程
1. **响应时限**：项目维护者（个人开发者）在收到举报 issues 后，将在 3 个工作日内确认接收，并在 issues 中告知处理大致时间线。
2. **调查过程**：维护者将查阅相关记录（如 GitHub 讨论日志、代码提交记录），必要时通过 issues 留言联系涉事方及证人核实情况，确保调查公正客观。
4. **结果反馈**：调查结束后，维护者将在 7 个工作日内通过举报 issues 反馈处理结果，说明处理依据及措施。

## 违规处理措施
根据违规行为的严重程度及影响范围，项目维护者（个人开发者）将采取以下处理措施，具体可灵活调整：
1. **书面警告**：针对单次轻微违规（如首次未注明来源、轻度言语不当），通过 GitHub 私信向违规者发出书面警告，明确整改要求及后续责任。
2. **临时限制权限**：针对多次轻微违规或首次较严重违规（如中度骚扰、提交低质量恶意代码），限制其 7-30 天内的代码提交、issues 讨论、评论等权限。
3. **临时封禁**：针对严重违规或屡教不改（如严重人身攻击、泄露核心机密），临时封禁其 30-90 天的项目访问权限，明确解封条件。
4. **永久封禁**：针对极其严重的违规（如恶意破坏项目、报复举报者、多次严重骚扰），永久封禁其项目所有访问权限，并在项目仓库公示（隐去隐私信息）。

## 监督
1. **持续优化**：根据参与者反馈及法律法规、行业规范的变化，对本准则进行修订，确保其适用性与有效性；修订过程将通过 GitHub issues 公开征求参与者意见，保障多元视角的融入。
2. **维护者自律**：作为个人开发者，将严格遵守本准则，主动接受社区监督，确保准则的公正执行。
